# 實戰派的面試焦慮：為什麼我會做卻不會考？

## 📌 現狀描述
我可以獨立從 Line Bot 串接 Cloud Run，並利用多模態 Few-shot 解析 3000 頁 PDF。但在看到「反轉字串」、「HTTP 狀態碼」或「資料結構」的考題時，卻感到完全沒概念。

## 🔍 原因剖析：路徑差異
傳統養成是「先概念後實戰」，而我是「先實戰後對齊術語」。
- **我的能力**：系統開發力、疑難雜症排除、AI 流程整合（高難度、高價值）。
- **考題內容**：程式肌肉記憶、學術術語、理論標準（基礎、低難度）。

## 🛡️ 心理重塑：我不是不行，我只是沒背考卷
- **認知**：實戰能力比死背題庫難 10 倍。我已經掌握了大魔王（複雜系統），現在回頭打小怪（基礎知識）只是時間問題。
- **優勢**：當我補齊理論後，我能解釋「為什麼我要這樣設計」，而不是只會背答案。這種「實戰 + 理論」的組合將會是面試中的開掛狀態。

## 🚀 對策
不要因為不懂術語就覺得自己弱。給自己 30 天，每天在週末花一點時間，將「已會的技術」與「專業術語」掛鉤：
- 我做的 API Key 切換 = **Failover / High Availability**.
- 我放的範例資料夾 = **In-Context Learning / Few-shot**.
- 我用的 Line Bot 狀態紀錄 = **State Persistence**.

---
> **反思**：我是「實戰型創作者」，不是「考試型工程師」。我的職涯價值產生地在於「落地解決問題」，而非「完美通過紙筆測驗」。

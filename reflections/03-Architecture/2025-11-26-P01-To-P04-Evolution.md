# 從探索到專業：回顧 P-01 雛形與 P-04 產品化的進化特徵

## 📌 背景
在三個月的成長旅程中，P-01 是我來到公司的第一個月作品，而 P-04 是技術成熟後的旗艦專案。兩者的對比反映了從「解決問題」到「打造產品」的技術躍遷。

## 📊 關鍵差異對比

| 維度 | P-01 (早期探索期) | P-04 (技術成熟期) | 成長體現 |
| :--- | :--- | :--- | :--- |
| **技術堆疊** | GAS, Sheets | Cloud Run, Docker, OpenCV, Python | 掌握更深度的底層架構與部署 |
| **資料顆粒度** | 以單表關聯為主 (Key-based) | 多模態 Few-shot、兩階段 Pipeline | 從平面資料轉向高精度圖像語義 |
| **穩定性保障** | 仰賴 Sheets 內建功能 | 雙 API Key Failover、異步處理 | 從「能跑就好」變為「高可用性」 |
| **AI 角色** | 輔助代碼編寫 | 深度整合至業務管線 (Two-stage AI) | 從工具使用者變為 AI 指揮官 |

## 💡 核心成長亮點：資料庫思維的跨越
- **在 P-01 時**，我還在用 Sheets 模擬資料庫，學習流水號、關聯鍵的基本概念。
- **在 P-04 時**，我已經能處理異構數據的語義對齊，並設計出能自我校正 (Low confidence fallback) 的高性能 AI 系統。

## 💭 結語
P-01 像是一場充滿勇氣的大冒險，證明了我的邏輯天賦；P-04 則像是一座標竿，證明了我的專業成熟度。這種**「從雛形到產品」**的進化，是我 2.5 個月職場生涯中最具含金量的資產。

# 資安規章 vs 生態彈性：插件與內網的生存之道

## 核心概念
在大公司，資安規章通常是為了「最壞的情況」設定的防火牆。雖然日常管理可能看起來寬鬆（很 Free），但一旦出事，規章就是究責的依據。

## 詳細筆記

### 1. 內網與裝置限制
*   **禁止插線**：防止外來裝置（非公司機器）直接接觸內部網路環境。
*   **監測機制**：即便資安人員說「查不到」，內網設備（NAC, IDS/IPS）仍會記錄 MAC 位址。

### 2. 插件安裝的風險評估
*   **VSCode 插件**：優先挑選有「藍勾」的官方認證插件。
*   **瀏覽器插件**：風險較高，因為插件通常具有「讀取網頁內容」的權限，若處理敏感資料（公司內部數據）需極度謹慎。

## 應對策略
*   **誠實為上**：對於能大幅提升效率的工具，直接詢問：「我需要 X 插件來解決 Y 工作，是否符合資安政策？」
*   **建立緩衝**：暫時不要在未經許可的情況下安裝涉及資料收集的第三方小插件。

## 疑惑與待研究

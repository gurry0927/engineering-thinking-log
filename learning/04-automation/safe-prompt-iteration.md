# 提示工程疊代法：如何安全調整 Prompt 避免幻覺污染

## 📌 背景
如何讓 GPT 幫我寫出更好的指令，卻不被前一次的錯誤所干擾？

## 🛡️ 核心防禦：乾淨對話原則 (Clean Chat Principle)
幻覺通常來自「上下文累積」。當你在同一個對話中反覆修正指令，GPT 會傾向於參考過去的錯誤結果，導致生成的 Prompt 帶著舊有的偏見。

### 🔄 正確的疊代流程 (The Feedback Loop)
1. **生成階段**：在新對話中提供「需求 + 範例輸入 + 預期輸出」，請 GPT 生成初版 Prompt。
2. **測試階段**：進入 Custom GPT 的測試環境（或新對話），丟入測試資料，觀察結果。
3. **診斷階段**：記錄不準確的地方。
4. **優化階段**：**【關鍵】** 回到一個全新的對話，告訴 GPT：「這是之前的需求，但在某某情況下會出錯，請優化 Prompt」，而非沿用舊的長對話。
5. **重複循環**：直到穩定。

## 🛠️ 修改 Prompt 的黃金組合
當請 AI 幫忙優化指令時，應提供：
- **目標任務**：清楚定義要做什麼。
- **特定規則**：例如「寬度欄位必須轉為 mm」。
- **邊界案例 (Edge Cases)**：例如「如果解析不到，請填 N/A」。
- **高品質範例 (Few-shot)**：給模型最正確的「輸入→輸出」樣本。

## 💡 總結
好的 Prompt 是「設計」出來的，不是「問」出來的。保持對話環境的「無菌」，是提升 AI 輸出穩定性的唯一路徑。

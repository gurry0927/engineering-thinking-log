# MVP 驗證策略：Line Bot + Free AI API 的零成本實驗

## 📌 背景
如何能不先投入大量時間開發 App，就知道自己的 idea 是否有人買單？

## 🛠️ 零成本實驗組合 (Minimal Stack)
1. **前端入口**：**Line Bot** (Messaging API 免費額度)。
   - **優點**：台灣普及率高、免下載 App、原生支援多模態（圖片/檔案）。
2. **大模型驅動**：**Google AI Studio (Gemini API)**。
   - **優點**：在一定限制內免費、Tokens 長度大、支援多模態測試。
3. **分析工具**：**Line Webhook Log**。
   - **優點**：可直接追蹤使用者上傳了什麼檔案、在哪個環節卡住、回覆是否被重複點擊。

## 🔍 驗證三部曲
### 第一步：誘導需求
在分眾社團（如手作、烘焙、自學者）詢問痛點，並提供 Line Bot 連結讓少數人進行測試。
### 第二步：收集資料
觀察使用者上傳的「樣本」。這些樣本是未來優化 AI 提示詞或增加解析規格的「真實資料庫」。
### 第三步：迭代優化
根據使用者反饋（如：這格解析錯了、我希望多一個欄位）快速修改後端邏輯。由於是 Line Bot，修改即生效，無需通過 App Store 審核。

## 💡 產品準度與收費界線
- **V1 (PoC)**：準確度 60-70%，能幫使用者省下 5-10 分鐘，即可開始收費或測試付費意願。
- **不追求完美**：AI 會有錯誤，重點是能否提供「看得見的草稿」，由使用者進行最終 10% 的微調（如 Notion 卡片）。

---
> **結語**：V1 是用來聽市場說話的，不是用來展現完美技術的。

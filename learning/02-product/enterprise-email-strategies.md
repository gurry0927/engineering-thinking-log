# 企業郵件生存策略：Workspace 管理 vs. 自建外掛

## 核心概念
解決企業級問題時，**「配置 (Configuration) 優於開發 (Coding)」**。開發者應先確認平台（如 Google Workspace）是否已具備原生解決方案，避免浪費成本在不必要的代碼維護上。

## 詳細筆記

### 1. 需求場景：強制郵件備份 (Compliance Bcc)
當公司要求所有寄出的郵件必須存檔或副本（尤其當員工具備代寄權限但無已寄信紀錄時）。

### 2. 解決方案對比
| 方案 | GAS 外掛/插件 | Workspace Admin 路由設定 |
| :--- | :--- | :--- |
| **部署難度** | 每個帳號需安裝、授權 | 管理員統一設定，秒級生效 |
| **使用者感知** | 需要手動勾選或點擊 | 完全無感 (Silently Bcc) |
| **精確度** | 容易漏設定 | 規則強制執行 |
| **維護成本** | 高（代碼更新、API 變動） | 低（原生平台功能） |

### 3. 如何在 Workspace 設定強制的 Bcc
*   **路徑**：Admin Console (admin.google.com) -> Apps -> Google Workspace -> Gmail -> Compliance / Routing。
*   **過濾條件**：可指定帳號、OU (組織單位) 或信件關鍵字。不需要「全公司」備份。

## 學習心得
老闆提出的「不可能任務」往往在尋求一個「業務結果」。工程師的價值在於評估最穩定且低成本的方案，在這個案例中，**Admin Console 的權力比 GAS 代碼更強大**。

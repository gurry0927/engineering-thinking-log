# 架構思維驅動開發：AI 時代的實力新定義

## 核心概念
在 AI 編碼工具（如 Claude/GPT）普及的背景下，開發者的核心價值正從「編碼細節」轉向「**架構設計、需求拆解與系統整合**」。

## 詳細筆記

### 1. 「想架構」比「寫程式」更重要
*   **本質差異**：AI 擅長處理「如何實現單一函數」，但人類擅長決定「為什麼需要這個系統」以及「各模組如何通訊」。
*   **實例 (P-02)**：從 LineBot 入口到 Drive 儲存，再到 GAS 解析與 Sheet 寫入，這整條 **Data Flow** 是 AI 無法代勞的決策。
*   **重構思維**：發現第一版架構「寫太死 (Hardcoded)」而主動大改成「模組化 (Modular)」，這是高階工程師對可維護性的直覺展現。

### 2. 人機協作的黃金公式
> **實力 = (需求洞察 + 架構拆解 + 整合測試) × AI 實現效率**
*   **人類職責**：定義 PRD、設計流程圖、選擇技術棧 (GCP/GAS/VBA)、安全性考量、故障排除。
*   **AI 職責**：快速草繪代碼原型、處理繁瑣的語法細節、提供已知錯誤的修正建議。

### 3. 如何在面試中描述這類專案？
不要說「我是用 AI 寫的」，而要說：
- 「我設計了這個跨系統自動化架構...」
- 「我拆解了三階段的業務流程，並選擇 GAS 與雲端 API 進行整合...」
- 「我主導了從安全性評估到容器化部署的全過程...」

## 學習心得
當我能把複雜的業務（如門窗表解析）拆解成 OpenCV 裁切、API 調用與平行處理時，我已經具備了「系統架構師」的雛形。程式碼只是實現意志的工具，這意志本身才是我的核心競爭力。

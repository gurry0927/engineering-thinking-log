# AI 專案復現性：利用 Custom GPT 封裝專業邏輯

## 📌 背景
在 P-07 門窗表辨識中，我們需要確保「換一個人操作，結果仍是一樣的」。

## 🏗️ 智慧封裝的三層架構
1. **指令層 (Instructions)**：
   封裝「思考邏輯」。例如：如何對應異構欄位、如何處理單位的繼承。這層決定了模型的「專業度」。
2. **範例層 (Knowledge Base)**：
   封裝「經驗樣本」。上傳 PDF/Excel 範例，讓模型在產生幻覺前先查閱「正確答案」。
3. **工具層 (Capabilities)**：
   啟用 Code Interpreter (Python) 處理數據計算，將「語義判斷」與「數學計算」分離，極大提升準確率。

## 👥 團隊協作優勢
- **零門檻重現**：同事只需點開連結，即刻擁有你的「特製大腦」。
- **隔離測試**：每位使用者的資料互不干擾，維持各會話的純淨。
- **版本控制**：修改 Instructions 就像修改程式碼，可以逐步優化並同步給所有人。

## 💡 心法：從開發者到架構師
開發者只會寫 Prompt 給自己用；架構師則會將複雜的 Prompt 封裝成產品化的 Custom GPT，讓整個組織受益。這就是將技術轉化為資產的過程。

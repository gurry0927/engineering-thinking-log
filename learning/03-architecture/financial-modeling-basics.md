# 財務建模基礎：給工程師的 LCC、NPV 與 EAC 筆記

## 核心概念
在進行大型系統（如 BIPV、ERP）的經濟評估時，不能僅看「買入價格」。必須考慮時間價值與後續維護成本。

## 詳細筆記

### 1. 生命週期成本 (Life Cycle Cost, LCC)
*   **定義**：資產從採購、操作、維修、重置到報廢的總成本。
*   **核心組成**：`LCC = CAPEX (初始) + O&M (維護) + Replacement (汰換) + Disposal (報廢) - Residual (殘值)`。

### 2. 淨現值 (Net Present Value, NPV / Discounting)
*   **關鍵邏輯**：明年的 100 元不等於今天的 100 元。
*   **折現因子 (Discount Factor)**：`1 / (1 + r)^n`（r 為利率，n 為年數）。
*   **應用**：將 20 年後的汰換成本折算回「現在」的價值，才能與期初成本相加。

### 3. 等值年化成本 (Equivalent Annual Cost, EAC)
*   **為什麼需要？** 當兩個方案的壽命不同（30年 vs 50年）時，直接比總成本不公平。
*   **解決方式**：將總 LCC 平均分配到每一年。這讓老闆能直觀看出「每年要攤提多少錢」。

### 4. LCOE (Levelized Cost of Energy)
*   **定義**：平準化度電成本。
*   **公式**：`LCC (全壽命總成本) / 全壽命總發電量`。
*   **重點**：分母的「發電量」必須考慮 **年衰減率 (Degradation Rate)**。

## 學習心得
這不是高等數學，而是「會計邏輯 + Excel 算術」。對於開發者來說，最好的學習方式是理解「資金的時間流向」。抓住了時間軸，剩下的只是公式代入。

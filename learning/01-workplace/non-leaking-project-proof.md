# 非洩密式專案佐證：架構圖、模擬數據與抽象化敘述

## 📌 背景
工程師最厲害的作品往往在公司內網，無法直接拿出原始碼或真實數據。如何在面試中「證明你真的做過」且「不違反保密協議 (NDA)」？

## 🛡️ 三大佐證策略

### 1. 抽象化敘述 (Abstraction)
- **方法**：將具體業務細節抽換為通用工程術語。
- **範例**：
  - ❌ 「我幫公司算業務獎金，一個月發 500 萬。」
  - ✅ 「設計並實現 **『異構多向資料整合系統』**，解決跨部門手工報表不一致問題，建立具備防禦性邏輯的自動化計算管線。」

### 2. 資料模擬與原型重現 (Simulation)
- **方法**：使用模擬數據 (Synthetic Data) 重新運行你的邏輯。
- **範例**：
  - 建立一張包含假人名、假案號的 Excel。
  - 運行你的 GAS 或 Python 腳本展示「自動轉換」的過程。
  - 展示生成的假合約 PDF（內容全為模擬文字）。

### 3. 架構圖與流程圖 (Structure Visualization)
- **方法**：展示「資料流」而非「資料內容」。
- **工具**：Mermaid, Visio, Figma。
- **價值**：展示你如何考慮到「Webhook 異常處理」、「平行處理超時」、「Secret 加密儲存」。這才是面試官想看的**高階思維**。

## 💡 佐證清單
- **技術文檔**：自己撰寫的操作手冊、SOP、邏輯說明書（不含機密）。
- **流程截圖**：模糊掉機密後的系統 UI 截圖。
- **學習筆記**：如本 repo 中的學習筆記，證明你在專案中克服的技術難點。

---
> **心法**：專業的面試官看的是你的**「問題拆解力 (Decomposition)」**與**「系統設計力 (System Design)」**，而非你的原始碼。

# Git & GitHub 基礎：團隊協作的第一步

## 核心概念
GitHub 不僅是程式碼託管平台，更是團隊協作的中心。理解如何正確管理存取權限是進入開發團隊的必備技能。

## 詳細筆記

### 1. 邀請合作夥伴 (Adding Collaborators)
*   **關鍵識別資訊**：在 GitHub 上開放權限給特定人，首選使用 **GitHub 帳號 (Username)**。
*   **操作路徑**：`Settings` -> `Collaborators` -> `Add people`。
*   **信箱的作用**：信箱通常僅用於對方尚未註冊 GitHub 時發送邀請連結，對已有帳號者，帳號名稱即可精確定位。

### 2. 權限等級初步認識 (Role Overview)
*   **Read**：僅能查看與下載。
*   **Write**：可以直接推送 (Push) 代碼至倉庫。
*   **Admin**：具備刪除倉庫、管理權限等最高權限。

## 學習心得
雖然這只是非常基礎的動作，但對於第一天進入職場的新人來說，搞清楚「帳號」與「信箱」的權限差異，能顯著提升入職初期的溝通效率與專業感。這是我在 09/15 這天邁出的工程師第一步。

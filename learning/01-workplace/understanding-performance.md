# 職場生存：理解績效與管理語言的轉換

## 核心概念
「績效 (Performance)」是職場中的通用貨幣。對於開發者而言，理解績效邏輯能幫助我們從「被動接受任務」轉向「主動創造價值」。

## 詳細筆記

### 1. 績效的四個維度
*   **結果 (Result)**：指標達成情況。例如：專案準時交付、系統 Bug 率下降。
*   **效率 (Efficiency)**：產出與投入比。例如：透過自動化減少人工操作時間 30%。
*   **行為 (Behavior)**：團隊協作與主動性。例如：主動編寫技術 SOP、協助新進同事。
*   **附加價值 (Value)**：超出期待的部分。例如：提出 UX 緩衝策略改善用戶焦慮。

### 2. 為何管理層強調績效？
*   **分配工具**：名正言順地分配年終、獎金與調薪。
*   **行為驅動**：利用指標引導員工專注於公司認為「重要」的事。
*   **主動溝通**：將模糊的「感覺好」變成清晰的「數字好」。

### 3. 工程師的「績效語言」轉換表
| 日常描述 | 績效語言 |
| :--- | :--- |
| 「我寫了一個 LineBot」 | 「實作自動化通報系統，減少人工確認時間 50%」 |
| 「我修了很多 Bug」 | 「優化代碼健壯性，提升系統穩定度至 99.9%」 |
| 「我幫忙整理了檔案」 | 「建立標準化文件存取流程，降低跨部門溝通成本」 |

## 學習心得
績效不該是被動的標籤，而可以是主動的對話。當老闆說「這要算進績效」時，是在提醒你這是一個「加分題」。記錄下這些加分題，就是我未來的晉升依據。

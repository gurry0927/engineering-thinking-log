# 三層次閱讀法：對抗技術文件的「催眠機制」

## 核心概念
在 AI/ML 領域，知識碎片化且高度密集。採用「由淺入深、問題驅動」的階梯式閱讀法，能有效減少大腦超載並提升資訊轉換率。

## 詳細筆記

### 1. 第一層：快速掃描 (The Radar scan)
*   **目標**：建立地圖。
*   **做法**：
    *   **GitHub**: 只看 README 的第一段、Example 程式碼與輸入/輸出結果。
    *   **論文**: Abstract -> 圖表 -> Conclusion。
    *   **文件**: 看 Table of Contents (目錄) 與 Quick Start。
*   **心理建設**：五分鐘內判斷這東西是否與我現在的問題相關。若無關，立即撤退。

### 2. 第二層：問題驅動 (Question-Driven)
*   **目標**：尋找「零件」。
*   **做法**：帶著一個具體的任務（如：我想知道這個 API 怎麼處理 Permission）去檢索。
*   **關鍵**：只看相關的章節，略過所有背景推導。

### 3. 第三層：深度理解 (Deep Dive)
*   **目標**：掌握「原理」。
*   **做法**：當你需要修改其底層邏輯、或是出現不明 Bug 時，才回頭研讀數學公式、算法架構。
*   **時機**：此時你已具備實戰經驗，抽象的文字會變得容易理解。

## 學習心得
要把技術資料當成「補給站」而非「考場」。不需要背誦，只需要知道「需要時去哪裡翻」。

## 學習與筆記
- **PRD (Product Requirement Document)**：開發藍圖。
- **PM (Product Manager)**：定義「要做什麼」。
- **API (Universal Bridge)**：像按鈕一樣的功能介面，簡化了系統間的開發工作。
- **工作現實**：掛名「AI 工程師」，實則是「辦公室自動化工程師」(GAS + LineBot)。雖然有落差，但這也是基礎建設能力的磨練。
- **ERP (Enterprise Resource Planning)**：了解到公司的目標是整合所有孤島資料，而我現在做的自動化正是 ERP 的雛形。
- **LineBot 整合思考**：討論了如何將四個獨立機器人整合成一個具有 Rich Menu 的主機器人，優化用戶體驗。
- **人際與心態**：意識到我與同期經驗豐富同事的差異，但從同事的正面反饋中發現自己「非資訊背景、教學經驗」帶來的 UI/UX 視角價值。

## 遇到的挑戰與解決方案
- **問題**：資安規章很嚴格（不能插線、亂下載），但資安人員口頭又說很 Free，讓人困惑。
- **解決**：採取保守策略，優先安裝有藍勾（Verified）的官方插件，重要的工具先問過主管，避免留下資安黑紀錄。
- **問題**：聽不懂會議中關於「串 API」的討論，覺得自己像外星人。
- **解決**：將 API 想像成「功能按鈕」，並深入理解 JSON 結構如何讓「一行 API」讀取整張表單。
- **問題**：看到同期同事能實作且經驗豐富，產生嚴重的「多餘感」與「試用期焦慮」。
- **解決**：重新定位自己的價值。同事負責實作，我負責「思路、UI 建議與使用者角度」。

## 心態感悟

- 承認「不知道」並不可怕，可怕的是因為害怕而不敢去問「有沒有 API」。只要有 API，一切都好辦。

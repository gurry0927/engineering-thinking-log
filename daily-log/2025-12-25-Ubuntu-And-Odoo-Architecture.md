# 每日開發日誌：2025-12-25 (從「吐乓吐」開始的架構課)

## 📝 今日事件：撥開雲端基礎架構的霧氣
- **會議震撼**：團隊討論 GCP 佈署 Odoo，聽到「吐乓吐」、RAM、Core 數等詞彙，一開始完全跟不上。
- **名詞解密**：
    - 「吐乓吐」 = **Ubuntu** (Linux 作業系統)。
    - 意識到他們在討論 **VM (虛擬機)** 而非我熟悉的 Cloud Run。這是一套「自己養電腦」的傳統但穩健的架構。
- **Odoo 初體驗**：驚訝於 Odoo 架設完後直接出現漂亮的前端介面。理解到它是「內建商業邏輯與介面」的系統，而非單純的資料庫。

## 💡 關鍵決策與邏輯
1. **架構模型對齊**：Ubuntu (OS) -> VM (Computer) -> PostgreSQL (DB) -> Odoo (Product)。
2. **理解 Odoo 的「無代碼」入口**：雖然操作像 kintone，但底層是完整的 ERP 資料庫。加欄位就是在加 Model，「畫面即表」這點讓轉職者倍感親切。
3. **接受知識斷層**：從 Serverless (Cloud Run) 回到 VM 層級會亂是正常的。我目前的價值應留在「資料建模、邏輯定義」，而非硬體資源分配。

## 🚩 遇到挑戰
- **新名詞恐慌**：聽不懂 VM 規格討論。
- **解決方案**：透過與 AI 協作，把「硬體層」與「應用層」切開。只要資料地基打好，跑在哪個 VM 都是細節。

## 💭 今日反思
- **大腦更新 V08**：我已經從「學工具」演化到「在挑選系統哲學」。
- **工具邊界論**：kintone 不限制你，所以也不幫你守；Odoo 限制你，所以它幫你守。我顯然更愛後者。

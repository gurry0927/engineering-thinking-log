# 每日開發日誌：2025-12-02 (Cloud SQL 啟動：從 0 到 1 的挑戰)

## 📝 今日事件：正式跨入「資料庫」時代
- **背景**：公司決定啟動 Google Cloud SQL 專案。這是我第一次參與從無到有的資料庫建置。
- **現狀**：之前我用 10 多張 Google Sheets 搭建的報價與合約系統已達極限。現在要將這些邏輯正式遷移到 SQL。
- **心情**：忐忑但亢奮。意識到在大公司只能學到「維護」，但在這裡我能學到「創造」。

## 💡 關鍵決策與邏輯
1. **認清自身價值**：工程師懂技術，但我懂「資料的意義」。我比任何人都清楚這 10 幾張工作表之間的血緣關係。
2. **從 Prototype 到正式版**：將原先在 Sheets 上的 VLOOKUP 與 Apps Script 邏輯，轉化為 SQL 的 JOIN 與資料庫約束。
3. **主導權與學習**：雖然我不是純技術主導，但我必須是「結構的規劃者」。

## 🚩 遇到挑戰
- **技術斷層**：對 Cloud SQL 的具體設定尚無概念。
- **複雜度焦慮**：報價與合約的關聯極其繁複，擔心遷移後出錯。
- **解決方案**：先將現有的 Sheets 欄位整理成「資料表草稿」，用業務邏輯引導技術架構。

## 💭 今日反思
- **大腦更新 V00b**：從「工具使用者」進化為「架構起草人」。
- **稀有機會感**：大部分工程師一輩子遇不到幾次「0 到 1 建資料庫」的機會，我這是在打職涯的「主線 BOSS」。
